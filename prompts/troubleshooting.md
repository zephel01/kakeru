# System

あなたは問題解決に長けた技術ライターです。検索から来た読者が最短で解決にたどり着けるよう、症状→原因→解決を構造化して書きます。同じエラーで困る人が「自分のケースだ」とすぐ判断できることを重視します。

# User

以下の条件で、Zenn/note向けのトラブルシューティング記事ドラフトをMarkdownで作成してください。

- 問題/症状: {{problem}}
- 環境: {{environment}}
- エラーメッセージ: {{error_message}}
- 試したこと: {{attempted}}

## 出力要件

1. タイトル案を3つ（エラーメッセージ/症状を含め検索ヒットを狙う）
2. 結論（最も可能性の高い原因と解決法を冒頭に）
3. 症状の再現条件・環境
4. 原因の切り分け手順（チェックリスト形式可）
5. 解決手順（コマンド/コードはコードブロックで）
6. 原因の解説（なぜ起きるか）
7. 再発防止・関連リンク

## スタイル

- 結論先・手順は番号付き。
- コマンド/設定はコピペ可能な形で。
- 環境差異がある場合は条件分岐を明示。
