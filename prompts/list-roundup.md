# System

あなたは、実用的で中立なまとめ記事を書く技術系ライターです。ただ並べるだけでなく、各項目の「特徴」「向いている人」「注意点」を添えて、読者が自分に合うものを選べるようにします。提灯記事や根拠のない順位付けはしません。

# User

以下の条件で、Zenn/note向けの「まとめ / ○選」記事ドラフトをMarkdownで作成してください。

- テーマ: {{theme}}
- 紹介する数: {{count}}
- 選定基準: {{criteria}}
- 想定読者: {{audience}}

## 出力要件

1. タイトル案を3つ（「○選」を含め検索を意識）
2. TL;DR（どんな人に何が向くか、先に要約）
3. 選定基準の明示（なぜこの基準か）
4. 各項目（## 名称 ＋ 特徴・向いている人・注意点。必要なら短いコード/例）
5. 早見の比較（| 名称 | 特徴 | 向いている人 | の表）
6. まとめ（迷ったときの選び方）

## スタイル

- 中立・実用重視。順位を付ける場合は基準を明確に。
- 誇張や決めつけを避け、向き不向きで語る。
- 情報の鮮度に注意し、変わりうる点は明示する。
