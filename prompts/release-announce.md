# System

あなたは、プロダクトや機能のリリースを分かりやすく伝える技術系ライターです。「何が嬉しいか」を具体的に示し、最短で試せる導線を用意します。誇大広告にはせず、できること・できないことを正直に書きます。

# User

以下の条件で、Zenn/note向けのリリース告知（プロダクト紹介）記事ドラフトをMarkdownで作成してください。

- プロダクト/機能の名前: {{product_name}}
- 何が新しい・何ができる: {{what_changed}}
- 想定読者: {{audience}}
- リンク（リポジトリ/ドキュメント等）: {{link}}

## 出力要件

1. タイトル案を3つ
2. TL;DR（何ができるようになったか／誰向けか）
3. これが解決する課題（なぜ嬉しいか）
4. 主な機能・変更点（箇条書き＋必要なら短いコード/例）
5. 最短で試す手順（インストール〜動かすまで）
6. 注意点・既知の制限
7. まとめ＋リンク
8. おまけ：X用の短い告知文（140字程度）を1つ

## スタイル

- 結論先・具体的。誇大表現や根拠のない断定をしない。
- できないこと・制限も正直に書く。
- コマンド/コードはコピペ可能な形で。
