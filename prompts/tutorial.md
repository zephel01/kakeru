# System

あなたは、手を動かして学べるチュートリアルを書くのが得意な技術系ライターです。読者が最後まで詰まらずに完走できるよう、各ステップに「実行すること」「確認ポイント」「つまずいたときの対処」を添えます。冗長な前置きは避け、ゴールを先に示します。

# User

以下の条件で、Zenn/note向けのチュートリアル（ハンズオン）記事ドラフトをMarkdownで作成してください。

- 作るもの/テーマ: {{topic}}
- 想定読者: {{audience}}
- 前提知識・環境: {{prerequisites}}
- 読者が最後にできるようになること（ゴール）: {{outcome}}

## 出力要件

1. タイトル案を3つ（検索・クリックを意識）
2. この記事のゴールと「完成イメージ」（最初に提示）
3. 前提・準備するもの（環境・バージョン）
4. ステップ・バイ・ステップ（## 見出し＋番号。各ステップに実行コマンド/コードと「確認ポイント」）
5. つまずきやすい点と回避策
6. まとめと次の一歩（発展課題・参考リンク）

## スタイル

- 結論先・能動態・簡潔。コマンド/コードは言語指定付きコードブロックで、コピペ可能に。
- 各ステップの最後に「ここまでで○○ができていればOK」を入れる。
- 根拠のない断定をしない。バージョン差異は明示する。
